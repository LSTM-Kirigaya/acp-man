# Feishu ACP Agent

基于 ACP 协议的飞书 Agent，连接 Kimi 和飞书平台，支持消息队列、多媒体处理和交互式卡片。

## 功能特性

- ✅ **ACP 协议连接**：通过 ACP 协议与 Kimi CLI 通信
- ✅ **消息队列**：确保消息按顺序处理，支持队列管理
- ✅ **多媒体处理**：支持图片、文件等媒体的存储和上下文传递
- ✅ **交互式卡片**：使用飞书卡片实现表单交互（如绑定路径）
- ✅ **私聊/群聊支持**：同时支持私聊和群聊场景
- ✅ **命令系统**：支持 `/bind`, `/status`, `/clear`, `/help` 等命令

## 多媒体处理流程

1. 用户发送图片/文件
2. 媒体文件下载并存储到本地
3. 媒体文件信息暂存在会话的 `pendingMedia` 列表中
4. 当用户发送新的文字消息时：
   - 清空 `pendingMedia` 列表
   - 将媒体文件作为上下文传递给 ACP
5. 新的对话开始时，媒体缓存会被清空

## 配置说明

### 环境变量

| 变量 | 必填 | 描述 |
|------|------|------|
| `FEISHU_APP_ID` | 是 | 飞书应用 ID |
| `FEISHU_APP_SECRET` | 是 | 飞书应用密钥 |
| `FEISHU_ENCRYPT_KEY` | 否 | 消息加密密钥 |
| `FEISHU_VERIFICATION_TOKEN` | 否 | 请求验证令牌 |
| `WEBHOOK_PORT` | 否 | Webhook 端口（默认 3000） |
| `WEBHOOK_PATH` | 否 | Webhook 路径（默认 /webhook/feishu） |
| `ACP_DEFAULT_CWD` | 否 | ACP 默认工作目录 |
| `KIMI_PATH` | 否 | Kimi CLI 路径 |
| `DEBUG` | 否 | 调试模式（true/false） |

### 飞书应用配置

1. 在飞书开放平台创建企业自建应用
2. 开启机器人功能
3. 获取 App ID 和 App Secret
4. 配置事件订阅：
   - 订阅事件：`im.message.receive_v1`
   - 请求地址：`https://your-domain.com/webhook/feishu`
5. 配置消息卡片（可选）：
   - 卡片请求地址：`https://your-domain.com/webhook/feishu`
6. 添加机器人到群聊或开启私聊权限

## 交互式卡片

### Bind 表单卡片

用户输入 `/bind` 命令后，机器人会发送一个交互式卡片：

```json
{
  "header": {
    "title": "🔗 绑定工作目录"
  },
  "elements": [
    {
      "tag": "input",
      "name": "bind_path",
      "placeholder": "请输入绝对路径"
    },
    {
      "tag": "button",
      "text": "确认绑定",
      "value": { "action": "bind_path_submit" }
    }
  ]
}
```

用户填写并提交后，表单数据通过卡片回调发送到 Webhook 服务器。

## 注意事项

1. **消息队列**：每个会话独立维护自己的消息队列，确保消息按顺序处理
2. **媒体缓存**：媒体文件默认保存 24 小时，超时自动清理
3. **会话超时**：长时间不活动的会话会被自动清理（默认 30 分钟）
4. **并发限制**：目前单条消息顺序处理，不支持并发响应
