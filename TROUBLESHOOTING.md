# 故障排查指南

## 问题：在飞书私聊中发送消息，控制台没有输出

### 检查清单

#### 1. 检查飞书应用权限

访问 [飞书开放平台](https://open.feishu.cn/) → 你的应用 → 权限管理

**必须开启的权限：**

| 权限 | 说明 |
|------|------|
| `im:message` | 接收用户发送的消息 |
| `im:message:p2p_msg:readonly` | **私聊消息（必须）** |
| `im:message:group_msg` | 群聊消息 |
| `im:message:send_as_bot` | 发送消息 |
| `im:message.reactions:write` | 表情回复 |

**检查方法：**
```bash
npm run diagnose
```

#### 2. 检查事件订阅配置

访问 [飞书开放平台](https://open.feishu.cn/) → 你的应用 → 事件订阅

**必须配置：**
- [x] 订阅方式：**使用长连接接收事件**（WebSocket）
- [x] 订阅事件：`im.message.receive_v1`

**配置步骤：**
1. 进入「事件订阅」页面
2. 在「订阅方式」中选择「使用长连接接收事件」
3. 在「事件订阅」中添加 `im.message.receive_v1`
4. 保存配置

#### 3. 检查应用发布状态

访问 [飞书开放平台](https://open.feishu.cn/) → 你的应用 → 版本管理与发布

**必须完成：**
- [x] 已创建版本
- [x] 已提交审核
- [x] 已发布（或已通过审核）

**注意：** 未发布的应用只能在测试企业中使用。

#### 4. 检查机器人配置

访问 [飞书开放平台](https://open.feishu.cn/) → 你的应用 → 机器人

**必须开启：**
- [x] 机器人能力已启用

#### 5. 检查私聊权限

**关键步骤：**

1. 在飞书客户端中找到你的机器人
2. 点击机器人的「配置」按钮
3. 确保「允许私聊」已开启

或者，在飞书开放平台：
- 机器人 → 消息设置 → 开启「支持私聊」

#### 6. 运行诊断脚本

```bash
cd /Users/kirigaya/project/acp-man
npm run diagnose
```

这个脚本会：
1. 测试 API 连接
2. 获取群聊列表
3. 启动 WebSocket 监听 10 秒
4. 提示你发送测试消息

**如果诊断脚本也没有收到消息，说明是飞书后台配置问题。**

### 常见问题

#### Q: 为什么 WebSocket 连接成功但没有收到消息？

**可能原因：**
1. **权限未开启** - 检查 `im:message:p2p_msg:readonly` 权限
2. **事件未订阅** - 检查 `im.message.receive_v1` 事件
3. **应用未发布** - 检查应用发布状态
4. **私聊权限未开启** - 检查机器人私聊设置

**排查步骤：**
```bash
# 1. 开启 DEBUG 模式查看详细日志
DEBUG=true npm run agent

# 2. 在另一个终端运行诊断脚本
npm run diagnose

# 3. 在飞书私聊中发送消息
# 4. 观察日志输出
```

#### Q: 如何确认事件订阅配置正确？

**检查步骤：**
1. 进入飞书开放平台
2. 找到你的应用
3. 点击「事件订阅」
4. 确认：
   - 订阅方式显示为「使用长连接接收事件」
   - 已订阅事件列表中有 `im.message.receive_v1`

#### Q: 如何确认权限配置正确？

**检查步骤：**
1. 进入飞书开放平台
2. 找到你的应用
3. 点击「权限管理」
4. 搜索以下权限，确认已添加：
   - `im:message`
   - `im:message:p2p_msg:readonly`
   - `im:message:send_as_bot`

#### Q: 应用已发布但还是收不到消息？

**可能原因：**
1. **缓存问题** - 飞书权限有缓存，等待 5-10 分钟
2. **企业审核** - 检查应用是否通过了企业管理员审核
3. **机器人未添加** - 确保机器人已添加到测试企业

### 快速修复步骤

1. **重新添加权限**
   - 删除所有权限
   - 重新添加必需的权限
   - 等待 5 分钟

2. **重新发布应用**
   - 创建新版本
   - 提交审核
   - 等待审核通过

3. **重启 Agent**
   ```bash
   npm run agent
   ```

4. **测试**
   - 在飞书私聊中发送消息
   - 观察控制台输出

### 仍有问题？

如果以上步骤都无法解决问题，请提供以下信息：

1. 运行 `npm run diagnose` 的输出
2. 运行 `DEBUG=true npm run agent` 的前 30 秒日志
3. 飞书应用的 App ID（前 8 位即可）
4. 飞书开放平台的截图：
   - 权限管理页面
   - 事件订阅页面
