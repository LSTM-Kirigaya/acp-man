# Feishu ACP Agent 架构设计

## 核心设计

### 1. 私聊 vs 群聊的区别处理

```
┌─────────────────────────────────────────────────────────────┐
│                        飞书消息                              │
│                   (私聊 / 群聊)                              │
└───────────────────────────┬─────────────────────────────────┘
                            │
            ┌───────────────┴───────────────┐
            │                               │
            ▼                               ▼
    ┌───────────────┐               ┌───────────────┐
    │    私聊       │               │    群聊       │
    │  (p2p)        │               │  (group)      │
    └───────┬───────┘               └───────┬───────┘
            │                               │
            ▼                               ▼
    ┌───────────────┐               ┌───────────────┐
    │  主 Agent     │               │ 检查是否绑定   │
    │               │               │               │
    │ • 无需绑定    │               │ 已绑定?       │
    │ • 默认路径    │           ┌───┴───┐           │
    │ • 直接回复    │           │       │           │
    └───────┬───────┘           ▼       ▼           │
            │               ┌─────┐  ┌─────┐       │
            │               │ 是  │  │ 否  │       │
            │               └──┬──┘  └──┬──┘       │
            │                  │        │          │
            ▼                  ▼        ▼          ▼
    ┌───────────────┐   ┌──────────┐  ┌──────────┐
    │  转发给 Kimi  │   │ 处理消息 │  │ 提示绑定 │
    │               │   │          │  │          │
    │ 使用默认路径  │   │ 使用绑定 │  │ /bind    │
    │               │   │ 路径     │  │ /path    │
    └───────────────┘   └──────────┘  └──────────┘
```

### 2. 群聊绑定流程

```
群聊首次使用
    │
    ▼
用户发送任意消息
    │
    ▼
┌─────────────────────┐
│ 检测到群聊未绑定     │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 发送绑定提示         │
│                     │
│ ⚠️ 此群聊尚未绑定    │
│ 请发送：             │
│ /bind /Users/xxx/xxx│
└──────────┬──────────┘
           │
           ▼
用户发送 /bind /path
    │
    ▼
┌─────────────────────┐
│ 保存绑定配置         │
│ config/bindings.json │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ ✅ 绑定成功！        │
│ 后续消息直接处理     │
└─────────────────────┘
```

### 3. 消息处理流程

```
收到消息
    │
    ▼
添加 🤔 表情（正在输入）
    │
    ▼
┌──────────────────────────┐
│ 判断聊天类型              │
└────────────┬─────────────┘
             │
     ┌───────┴───────┐
     │               │
     ▼               ▼
 ┌───────┐      ┌───────┐
 │ 私聊  │      │ 群聊  │
 └───┬───┘      └───┬───┘
     │               │
     ▼               ▼
 直接处理      检查绑定状态
     │               │
     │          ┌────┴────┐
     │          │         │
     │          ▼         ▼
     │      已绑定      未绑定
     │          │         │
     │          ▼         ▼
     │      处理消息   提示绑定
     │          │
     └────┬─────┘
          │
          ▼
    加入消息队列
          │
          ▼
    转发给 Kimi ACP
          │
          ▼
    等待 Kimi 回复
          │
          ▼
    发送回复给用户
          │
          ▼
    移除 🤔 表情
```

## 权限要求

### 必须权限
- `im:message` - 接收消息
- `im:message:send_as_bot` - 发送消息
- `im:message.reactions:write` - 表情回复（"正在输入"效果）
- `im:chat:readonly` - 读取群聊信息

### 可选权限
- `contact:user.base:readonly` - 获取用户基本信息

## 核心组件

### 1. ConfigManager
- 管理群聊绑定配置的持久化
- 保存路径：`./config/bindings.json`

### 2. ChatSessionManager
- 管理会话生命周期
- 处理消息队列
- 区分私聊/群聊逻辑

### 3. FeishuAcpAgent
- 主控制器
- 处理消息路由
- 协调各组件工作

### 4. ReactionManager
- 管理"正在输入"表情
- 添加/移除 🤔 表情

## 使用示例

### 私聊
```
用户: 你好
机器人: 你好！我是 Kimi...
（无需绑定，使用默认路径）
```

### 群聊（首次）
```
用户: 你好
机器人: ⚠️ 此群聊尚未绑定...
       请发送：/bind /Users/xxx/project

用户: /bind /Users/kirigaya/myapp
机器人: ✅ 绑定成功！

用户: 分析一下代码
机器人: 好的，我来分析...
（使用绑定的 /Users/kirigaya/myapp 路径）
```

### 群聊（已绑定）
```
用户: 重构这个函数
机器人: 我来帮你重构...
（无需 @机器人，直接处理）
```
